宏站基站

基站是什么？

基站是给手机（UE/MS）提供信号，宏站的基站是由BBU,RRU,天线组成的，一个宏站一般分为三个小区，三个扇区，宏站是负责室外覆盖的。

什么是天线的下倾角和方位角？

下倾角是天线的背面与抱杆之间的夹角，下倾角越大电磁波覆盖的范围越小信号强度越强，

方位角是天线的正面覆盖方向偏离正北的顺时针的角度

天线的驻波比

形容天线与馈线匹配的指标，大于3(完全不匹配)会产生驻波比报警，小于1.5为正常。

电磁波的特点（2个）

1、可以搭载信息

2[^1]、频率越高，波长越短,越趋近于直线传播(绕射能力越差).

[^1]:
频率越高，波长越短。 源自 C（光速） = 波长 * 频率 。
频率越高，传播距离越短。频率越低传播距离越远所以频率越低越好

1、增强室分的覆盖(主要目的)

2、分担室分的业务量

室分频点

中国移动使用的室分频点都是TDD的E频段，分E1E2E3，频点和中心频率分别是38950、39148、39292，针对这些频点有室分优先原则。

室分优先原则

90%以上区域信号强度不低于-105，SINR大于等于3，满足此条件情况下，手机应优先占用室分频点小区。

室分优先怎么体现

通过不同频点互操作参数体现的：

空闲状态下，室分频段的优先级是4G里最高的为7，有室分信号的情况下优先重选到室分站点

连接态，D/A/F/FDD1800忘E频段的切换事件设置为A4快切事件，只需要室分的邻区信号强度大于A4门限，即可上报A4切换事件

什么是室分外泄

  1. 室内覆盖的信号泄漏到室外就是室分外泄

  2. 室分外泄的原则：要求室外10米处应满足RSRP小于110dBm，或室内小区外协的RSRP比室外主小区的RSRP低10dB(当建筑物距离道路不足10米时，以道路靠建筑物一侧作为参考点)

FDD与TDD的优缺点对比

FDD的优点：

  1. TDD下载速度天然不如FDD

  2. FDD信号质量比TDD好因为上下行分段干扰少

TDD的优点：

  1. 成本低

  2. 灵活的支持分对称业务

  3. 节省频段资源，TDD频段资源占用比FDD频段资源占用小

关键技术

**MIMO(多天线技术)**：小区覆盖与容量增益的最重要的来源； Mode 传输模式 技术描述 天线要求 场景
TM1 单天线传输 单天线 单天线 无法放双通道室分系统的室内站
TM2 发射分集 多天线发同数据的副本 大间距多天线 信号质量不好的如小区边缘
TM3 开环空间复用 不反馈信道信息多发多收 大间距多天线 信号质量高SINR高的支持高速移动
TM4 闭环空间复用 反馈信道信息多发多收 大间距多天线 信号质量高SINR高的不支持高速移动

**OFDMA(正交频分复用)**：更好的支持更大的带宽(极大的提高了频段的利用率，只在下行波)

**CA(载波聚合)**：载波聚合技术使得系统可以获得更大的系统带宽，实现更好的用户高速业务体验

**AMC(自适应调制解码)**：充分利用信道质量，引入高阶调制，提升小区容量

物理信道^13

**小区搜索**波及的物理信道：SCH -\> PBCH -\> PCFICH -\> PDCCH -\>PDSCH(获取DBCH)

**随机接入**波及的物理信道：PRACH -\> PCFICH -\> PDCCH -\> PDSCH -\> PUSCH

**下行数据**波及的物理信道：PCFICH -\> PDCCH -\> PDSCH -\> PUCCH

**上行数据**波及的物理信道：PCFICH -\> PDCCH -\> PUSCH -\> PHICH
下行物理信道与信道名称 功能简介
PBCH[^14] 广播信道 用于承载系统广播信息、MIB广播信息
PDSCH[^15] 下行共享数据信道 用于承载下行用户数据
PDCCH[^16] 下行控制信道 用于上下行调度、功控等控制信令的传输
SCH[^17] 同步信号 用于时频同步和小区搜索
PRACH[^18] 随机接入信道 用于用户随机接入请求信息和上行同步
PUSCH[^19] 上行共享数据信道 用于承载上行用户数据
PUCCH[^20] 上行公共控制信道 用于HARQ反馈、CQI反馈、调度请求指示等L1，L2控制信令(上行控制信道)

[^14]: PBCH(Physical broadcast channl)
只传播MIB消息，MIB是下行链路的系统带宽.PHICH配置信息.系统帧号。

[^15]: PDSCH(Physical Downlink Shared Channel) 用于承载下行用户数据。也有SIB。

[^16]: PDCCH(Physical downlink control channel)

[^17]: SCH(Synchronization signal)

[^18]: PRACH(Physical Random Access Channel)

[^19]: PUSCH(Physical Uplink Shared Channel)

[^20]: PUCCH(Physical Uplink Control Channel)

RS参考信号和MOD3干扰

PCI的mod3的值会影响RS在RB中的分布位置，如果同频小区间分布位置重合会造成mod3的干扰

手机开机入网的流程

小区搜索：

第一步：UE解调主同步信号PSS，实现符号同步，获得小区组内序号123中的一个。

第二步：UE解调辅同步信号SSS，实现帧同步，获得CP长度和小区组ID，结合组内序号，获得小区的PCI。

系统消息接收

接收的系统消息广播有MIB+SIB+SI
其中MIB是PBCH传播的，SIB是从BCCH到DL-SCH到PDSCH.

小区选择：

在每一个频率上，UE只需要搜索信道质量最好的小区，一旦一个合适的小区出现就是满足s准则S>0的小区，UE选选择他进行驻留

S准则

Srxlev=测量小区的RSRP值 –
（小区的RSRP最小接收强度+最小小区接收强度的偏置值）–（Pmax小区允许UE的最大上行发射功率—UE自身能力决定的最大上行发送功率一般为0）

随机接入：^21

1. RRC空闲态初始接入     竞争

2. 无线链路断开接入     竞争

3. 切换时的随机接入     竞争非竞争都行

4. 上行失步        竞争

5. 下行失步     竞争非竞争都行

只有随机接入完成后，手机和基站才能正常进行数据互操作。目的：取得手机与基站的上行同步和申请上行资源。

竞争和非竞争

竞争接入就是UE发出前序列希望得到基站的资源授权，但是有可能有多个用户同时发送，基站不知道是谁发的，所以需要后续各个UE发一条独一无二的消息MSG3，如果基站同意回传MSG4。触发条件：UE初始接入，UE重建，上行失步，无SR资源，切换。

非竞争接入：就是基站提前授权资源，无需竞争。触发条件：切换，有下行数据发送但上行失步，定位。

ATTACH：

完成UE在网络中的注册，完成核心网(EPC)对改UE默认承载的建立

ATTACH的信令流程

RRC连接：就是建立SRB1的过程 就是上图③-④的过程。

建立流程

1. RRC ConnectionRequest
建立请求在CCCH逻辑信道发送，承载信令的是SRB0,并启动T300(T300计时器是RRC建立请求到建立SRB1的时间
超过则失败 常设200ms)。

2. RRC ConnectionSetup 在CCCH逻辑信道发送，承载信令的是SRB0 MSG4建立SRB1
UE收到MSG4后停止T300.

3. RRC ConnectionReject 拒绝RRC连接
开始T302计时（T302是拒绝后再次申请RRC连接要等待的时间 常设值16s
在时间内小区禁止接入）

4. RRC ConnectionSetupCommplete 在DCCH 逻辑信道发出 信令承载的SRB1,
RRC建立完成。

系统消息

MIB：在PBCH上发送，MIB上传输重要参数有：下行链路系统带宽；PHICH配置信息；系统帧号；

SIB：SIB = SIB1+SIBs SIBs = SIB2 - SIB8

SIB1包含：一个或者多个PLMN标识;Track area code(TAC号);小区ID；

SIB2 小区的无线配置，其他基本配置。

SIB3 小区重新的信息，主要关于服务小区，

SIB4 频内邻区列表，黑白名单

SIB5 频内邻区列表。

SIB6-8 2G 3G 4G.

掉线的定义

LTE系统中，UE处于连接状态下如果发生切换失败、无线链路失败、地城完整性保护失败和RRC重新配置失败后UE会进行RRC连接重建^23，如果RRC连接重建失败，UE将转入RRC-LDLE状态，发生掉线。

侦听寻呼信道

进行邻区测量和小区重选

TAU（位置更新）

获得系统消息

启用DRX省电（非连续接收）

RRC CONNECTED 连接态

上下行数据传输

切换

提供信道质量和反馈信息

RRC重建

1.
开始RRC重建进行小区选择，启动T311定时器（T311是重建开始后进行小区选择的时间
失败就会进入空闲态掉线 常设值10s）。

2.
选择小区成功后停止T311发生重建请求启动T301（T301是重建RRC连接的时间和300差不多

3. 收到回应后停止T301回复重建完成 进入连接态。
